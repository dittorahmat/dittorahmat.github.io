# System Patterns

## System Architecture:
- Static website generated by Hugo
- Hosted on GitHub Pages
- Content written in Markdown
- Theme-based design

## Key Technical Decisions:
- Using Hugo for static site generation due to speed and flexibility.
- Hosting on GitHub Pages for free and easy deployment.
- Markdown for content creation for simplicity and portability.

## Design Patterns:
- Content-driven architecture
- Theme-based presentation
- Modular design for content sections (portfolio, blog, about)

## Component Relationships:
- Hugo processes Markdown content and theme templates to generate HTML, CSS, and JavaScript.
- GitHub Pages serves the generated static files.
- User interacts with the website through a web browser.
